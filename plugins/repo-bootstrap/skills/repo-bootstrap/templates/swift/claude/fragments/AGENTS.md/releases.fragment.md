**Releases.** Tagging `v*` triggers `.github/workflows/release.yml`, which forwards to the shared `release-swift.yml@swift-v1` reusable workflow: a universal (arm64 + x86_64) `swift build`, a GitHub release, and a Homebrew binary cask pushed to `{{GITHUB_USER}}/homebrew-tap`. The version comes from the tag, and the executable product must keep the repo's name — that's the whole calling contract. The release refuses to run unless the tagged commit is on `main` — tag a merged commit (e.g. `git tag vX.Y.Z origin/main`), not a feature branch. One-time setup requires `HOMEBREW_TAP_TOKEN` plus all five `MACOS_*` repository secrets. The binary is Developer-ID-signed and notarized; missing credentials fail the release before publication (see `reference/swift-ci-and-release.md`).
