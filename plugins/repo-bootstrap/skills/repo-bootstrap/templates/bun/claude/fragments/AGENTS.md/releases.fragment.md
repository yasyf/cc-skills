**Releases.** Tagging `v*` triggers `.github/workflows/release.yml`, which forwards to the shared `release-bun.yml@bun-v1` reusable workflow: a per-target `bun build --compile` on native runners (macOS + Linux, arm64 + x86_64 — platform-native deps rule out cross-compiling), a GitHub release with the four zipped binaries, and a Homebrew binary cask pushed to `{{GITHUB_USER}}/homebrew-tap`. The version comes from the tag; the entry point is `src/index.ts`, the compiled binary keeps the repo's name, and the bun toolchain is pinned by the repo's `.bun-version` file — that's the whole calling contract. The release refuses to run unless the tagged commit is on `main` — tag a merged commit (e.g. `git tag vX.Y.Z origin/main`), not a feature branch. One-time setup requires `HOMEBREW_TAP_TOKEN` plus all five `MACOS_*` repository secrets. The macOS binaries are Developer-ID-signed and notarized; missing credentials fail the release before publication (see `reference/bun-ci-and-release.md`).
