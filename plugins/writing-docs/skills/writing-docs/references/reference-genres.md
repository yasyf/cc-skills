# Reference genres: cheat sheets and examples catalogs

Two high-value page formats that don't get their own Diataxis mode but earn a skeleton of their own. A cheat sheet is a format of **reference**; an examples catalog entry is a format of **how-to**. Naming the shape makes it repeatable; the page still lives inside its parent mode's rules.

## Mode purity: a cheat sheet is not a lite reference

A cheat sheet stays pure reference: facts, scannable, no narrative, no step-by-step. A catalog entry stays pure how-to: one runnable task, no concept teaching. The moment a catalog entry starts explaining *why*, it has bled into explanation — cut that and link out.

Keep each genre distinct from its plain cousin. A cheat sheet is not a "lite reference" and a reference is not a "verbose cheat sheet" — never half-label one as the other. A page that is exhaustive and signature-complete is reference; title it as reference. A page that is one scannable line per entry is a cheat sheet; put it where readers reach for speed. A page that is half tables and half copy-lines, titled "X cheatsheet" but built like a reference, serves neither reader. Pick one identity per page, and make the title tell the truth about it.

## Cheat sheet

A one-screen scan sheet for a reader who knows the tool and wants the snippet, now. ast-grep's cheat sheets are the model.

- Title naming the surface ("Hooks", "Matchers")
- One framing sentence, then straight into entries
- Entries grouped by **workflow or intent**, not alphabetically: a heading per group, with a category emoji that anchors the eye (🛑 block, 💬 nudge, 🔍 lint)
- Each entry: a bold name, a 5-10 word description, and one real copy-paste line in a fenced block
- Searchable text, never an image — a screenshot or PDF cannot be searched or copied
- A "See also" line to the how-to that teaches the surface and the reference that lists every option

The cheat sheet shows the *common* form of each thing; the reference shows *every* form. They cross-link, and neither carries the other's depth.

## Examples catalog entry

One runnable specimen per page, framed as a problem the reader has hit. ast-grep's catalog is the model: every entry is copy-paste-complete and one step from running.

- Title naming the capability ("Command safety")
- A one-line capability-and-problem statement, optionally a capability badge (what it does, how it's verified)
- A short problem paragraph: the situation the reader is in
- The complete code, single-sourced and embedded, never a retyped fragment
- The input it acts on, shown — the cases it catches and the cases it allows, derived from the example's own tests so the page cannot drift
- A before/after diff or the expected output, generated by running it
- A one-line "run it yourself" with the exact command
- A "See also" to the how-to and reference behind it

Order a catalog simple to complex and group it by job. Every entry stands alone — no entry depends on reading the one before it.
